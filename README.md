# 🏥 Hospital Appointment and Payment System

A full-stack web application designed to streamline hospital appointment booking and payment processes. This system digitizes traditional hospital workflows, reducing manual effort and improving patient experience.

---

## 🚀 Tech Stack

- **Frontend:** React.js  
- **Backend:** ASP.NET Core Web API  
- **Database:** SQL Server  
- **ORM:** Entity Framework Core  
- **Authentication:** JWT (JSON Web Token)  
- **Payment Gateway:** Razorpay  
- **Other:** Email Service Integration  

---

## 📌 Project Overview

I developed this project to solve real-world problems faced in many hospitals where appointment booking is still manual. This often leads to long waiting times, poor scheduling, and inconvenience for patients.

This system provides a **digital solution** that improves efficiency, reduces waiting time, and enhances the overall user experience.

---

## ✨ Key Features

- 🔐 Secure User Authentication (JWT आधारित)
- 📅 Online Appointment Booking System
- 💳 Razorpay Payment Integration
- 📧 Email Notifications (confirmation, updates, rescheduling)
- 🔄 Smart Slot Management System
- 💸 Refund System for cancellations
- 👨‍⚕️ Doctor Management (Admin side)
- 📊 Admin Dashboard for appointment control

---

## 👤 User Flow

1. User registers and logs in securely  
2. Selects doctor, date, and time slot  
3. Adds problem description  
4. Makes payment via Razorpay  
5. Appointment gets confirmed after successful payment  

---

## 🛠️ Admin Features

- Secure Admin Login  
- View all appointments  
- Accept / Reject / Mark Pending  
- Add and manage doctors  
- Manage scheduling  

---

## 🔄 Smart Slot Management

If a user cancels an appointment:
- The slot becomes available  
- Another waiting user can be assigned  
- Email notification is sent for early slot availability  

---

## 💸 Payment & Refund System

- Secure online payments using Razorpay  
- Automatic refund handling for cancellations or rejections  

---

## 📧 Notification System

Automated emails are sent for:
- Appointment confirmation  
- Status updates  
- Rescheduling alerts  

---

## 🧩 Project Modules

- **User Module:** Registration, login, booking, payments  
- **Admin Module:** Appointment management, doctor management  
- **Doctor Module:** Doctor details and availability  
- **Payment Module:** Transactions and refunds  
- **Notification Module:** Email alerts  

---

## 🏗️ Architecture

The project follows a **Clean Architecture** approach:

- Controller Layer  
- Service Layer  
- Repository Layer  

This ensures:
- Scalability  
- Maintainability  
- Separation of concerns
